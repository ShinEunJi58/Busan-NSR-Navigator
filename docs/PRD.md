## 📋 프로젝트 개요

### 문제 정의
중소기업이 북극항로(NSR) 활용을 고려할 때:
1. **정보 부족**: 복잡한 물류 데이터를 이해하기 어려움
2. **전문성 부족**: 24/7 물류 전문가 상담 불가능
3. **의사결정 지연**: 수에즈 vs NSR 비교 분석에 시간 소요

### 해결책
**"AI 에이전트가 24/7 실시간으로 북극항로 관련 모든 질문에 답변하고, 데이터 기반 의사결정을 지원하는 통합 플랫폼"**

### 핵심 가치
- 🤖 **AI 상담**: RAG 기반 실시간 질의응답
- 📊 **데이터 분석**: 경제성/리스크 자동 진단
- 💡 **맞춤 추천**: 기업별 최적 품목 제안
- 🔒 **로컬 실행**: 데이터 보안 + 무료 운영

## ⚙️ 핵심 기능 (4가지)

### 1. 🤖 AI 물류 상담 에이전트 (RAG 기반) ⭐ 주 기능

**목적:** 중소기업이 언제든 북극항로 관련 질문을 하면 AI가 즉시 답변

**기술 스택:**
- **LLM:** Ollama (로컬 실행, 무료)
  - 모델: `llama3.2:3b` (한국어 지원, 가벼움)
- **RAG:** LangChain + ChromaDB (벡터 DB)
- **문서:** Markdown/PDF 형태의 북극항로 가이드

**작동 방식:**
```
사용자 질문: "겨울에도 북극항로를 이용할 수 있나요?"
    ↓
[RAG 시스템]
1. 질문을 벡터로 변환
2. ChromaDB에서 관련 문서 검색
3. 검색된 문서를 컨텍스트로 LLM에 전달
    ↓
[Ollama LLM]
4. 컨텍스트 기반 답변 생성
    ↓
AI 답변: "겨울철(11-4월)에도 이용 가능하지만 
쇄빙선 에스코트가 필수입니다. 
비용은 여름 대비 약 20% 증가하며..."
```

**주요 기능:**
- ✅ 자연어 질의응답
- ✅ 문서 기반 정확한 답변 (환각 방지)
- ✅ 대화 히스토리 유지
- ✅ 출처 표시 (어느 문서에서 가져왔는지)

**데이터 소스:**
```
knowledge_base/
├── nsr_basic_guide.md          # 북극항로 기본 가이드
├── seasonal_operations.md      # 계절별 운항 정보

├── risk_factors.md             # 리스크 요인
```

### 2. 💰 경제성 진단 (Cost Calculator)

**목적:** 수에즈 vs NSR 비용/시간/거리 자동 비교

**입력:**
- 화물 컨테이너 수 (TEU)
- 목적지 (유럽 주요 항구)
- 출발 시기 (계절별 비용 차이)

**출력:**
- 비교표 (거리, 시간, 비용, CO₂)
- 시각화 그래프 (Plotly)
- 절감액 계산
- AI 추천 의견

**계산 로직:**
```python
# 기본 공식
총비용 = 기본운임 + (거리 × km당비용) + (TEU × 컨테이너비용) + 계절할증

# 계절 할증
여름(7-10월): 할증 없음
겨울(11-4월): +20% (쇄빙선 비용)

# CO₂ 계산
CO₂ = 거리 × 배출계수 × TEU
```

### 3. ⚠️ 리스크 모니터링 (Risk Radar)

**목적:** 북극항로 운항 리스크 실시간 모니터링

**모니터링 항목:**

   - 러시아 영해 통과 관련 이슈
   - 보험료율 변동
   - 제재 영향

3. **실시간 뉴스**
   - 북극항로 관련 최신 뉴스 (크롤링 또는 RSS)
   - AI 요약 제공

- AI가 최신 데이터를 분석하여 답변

## 🤖 AI 에이전트 상세 설계

### RAG 시스템 구조 (JavaScript)

```javascript
// js/ai-agent.js

class NSRAgent {
  constructor() {
    this.apiKey = 'YOUR_HUGGING_FACE_API_KEY'; // GitHub Secrets에서 주입
    this.model = 'google/flan-t5-large';
    this.apiUrl = `https://api-inference.huggingface.co/models/${this.model}`;
    this.knowledgeBase = null;
    
    this.init();
  }
  
  async init() {
    // 지식 베이스 로드
    const response = await fetch('data/knowledge.json');
    this.knowledgeBase = await response.json();
  }
  
  // 1. 키워드 검색
  searchKnowledge(question) {
    const keywords = this.extractKeywords(question);
    const relevantDocs = [];
    
    for (const doc of this.knowledgeBase.documents) {
      const score = this.calculateRelevance(keywords, doc.keywords);
      if (score > 0.3) {
        relevantDocs.push({ ...doc, score });
      }
    }
    
    // 점수 순으로 정렬, 상위 3개 반환
    return relevantDocs
      .sort((a, b) => b.score - a.score)
      .slice(0, 3);
  }
  
  extractKeywords(text) {
    // 간단한 키워드 추출 (불용어 제거)
    const stopWords = ['은', '는', '이', '가', '을', '를', '에', '의'];
    return text
      .split(/\s+/)
      .filter(word => !stopWords.includes(word))
      .map(word => word.toLowerCase());
  }
  
  calculateRelevance(queryKeywords, docKeywords) {
    let matches = 0;
    for (const qk of queryKeywords) {
      for (const dk of docKeywords) {
        if (qk.includes(dk) || dk.includes(qk)) {
          matches++;
        }
      }
    }
    return matches / queryKeywords.length;
  }
  
  // 2. AI 질의응답
  async ask(question) {
    try {
      // 관련 문서 검색
      const relevantDocs = this.searchKnowledge(question);
      
      // 컨텍스트 구성
      const context = relevantDocs
        .map(doc => doc.content)
        .join('\n\n');
      
      // 프롬프트 구성
      const prompt = `
당신은 KOTRA의 북극항로(NSR) 전문 상담 AI입니다.

참고 문서:
${context}

질문: ${question}

위 참고 문서를 바탕으로 정확하고 친절하게 답변해주세요. 
답변은 2-3문장으로 간결하게 작성하세요.

답변:`;
      
      // Hugging Face API 호출
      const response = await fetch(this.apiUrl, {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.apiKey}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          inputs: prompt,
          parameters: {
            max_length: 200,
            temperature: 0.7
          }
        })
      });
      
      const result = await response.json();
      
      return {
        answer: result[0].generated_text,
        sources: relevantDocs.map(doc => ({
          title: doc.title,
          file: doc.id
        }))
      };
      
    } catch (error) {
      console.error('AI 에러:', error);
      return {
        answer: '죄송합니다. 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요.',
        sources: []
      };
    }
  }
}

// 사용 예시
const agent = new NSRAgent();
const response = await agent.ask('겨울에도 북극항로를 이용할 수 있나요?');
console.log(response.answer);
console.log(response.sources);
```

### 1단계: Ollama 설치 (Windows)
```bash
# Ollama 설치
winget install Ollama.Ollama

# 모델 다운로드 (약 2GB, 5분 소요)
ollama pull llama3.2:3b

# 테스트
ollama run llama3.2:3b "안녕하세요"
```

### 2단계: Python 환경 설정
```bash
# 가상환경 생성
python -m venv venv

# 활성화
venv\Scripts\activate

# 패키지 설치
pip install -r requirements.txt
```

### 3단계: 지식 베이스 초기화
```bash
# AI 에이전트 초기화 (벡터 DB 생성)
python src/ai_agent.py --init
```

### 4단계: 앱 실행
```bash
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 자동 열림

## 📈 면접관 예상 질문 & 답변

### Q1: "왜 Ollama를 선택했나요?"
**A:** "세 가지 이유입니다. 첫째, 완전 무료로 API 비용이 없습니다. 둘째, 로컬에서 실행되어 중소기업의 민감한 데이터가 외부로 유출되지 않습니다. 셋째, 인터넷 없이도 작동하여 안정성이 높습니다. 실제 KOTRA 배포 시에도 내부망에서 안전하게 운영할 수 있습니다."

### Q2: "RAG가 뭔가요?"
**A:** "Retrieval-Augmented Generation의 약자로, LLM이 답변하기 전에 관련 문서를 먼저 검색하여 참고하는 기술입니다. 이를 통해 AI가 잘못된 정보를 만들어내는 '환각' 현상을 방지하고, 정확한 답변만 제공할 수 있습니다."

### Q3: "실제 배포는?"
**A:** "현재는 로컬 PC에서 실행되지만, Docker 컨테이너로 패키징하여 KOTRA 내부 서버에 배포 가능합니다. Streamlit Cloud나 Hugging Face Spaces를 사용하면 외부 공개도 가능합니다."

### Q4: "개발 기간은?"
**A:** "핵심 AI 에이전트는 6시간, 전체 통합은 12시간 소요되었습니다. LangChain과 Streamlit의 강력한 라이브러리 덕분에 빠른 개발이 가능했습니다."

### Q5: "확장 가능성은?"
**A:**
- 더 큰 LLM 모델 (llama3:70b) 사용
- 실시간 뉴스 크롤링 자동화
- 다국어 지원 (영어, 중국어)
- 사용자별 맞춤 추천
- KOTRA 바우처 시스템 연동
